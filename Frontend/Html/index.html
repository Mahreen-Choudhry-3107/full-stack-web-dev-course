1. The HTML document · skeleton

Every HTML page starts with <!DOCTYPE html> and the <html> root. Inside we have <head> (metadata, title) and <body> (visible content).

<!-- ========== BASIC TEMPLATE ========== --> <!DOCTYPE html> <!-- tells browser this is HTML5 --> <html lang="en"> <!-- root element, language attribute --> <head> <meta charset="UTF-8"> <!-- character encoding (UTF-8) --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>page title</title> <!-- appears in browser tab --> </head> <body> <!-- all visible content goes here --> </body> </html>

element Tags are the building blocks. Most have an opening <tag> and closing </tag>. Some are self-closing: <img>, <input>, <br>.

2. Text basics · headings & paragraphs

Headings h1 → h6

<h1>Main title (only once per page)</h1> <h2>Section heading</h2> <h3>Sub-section</h3> <h4>Sub-sub</h4> <h5>level 5</h5> <h6>level 6</h6>

Paragraphs & inline text

<p>This is a paragraph. It has space above and below.</p> <p>Use <strong>strong</strong> (bold) and <em>emphasis</em> (italic). <br> <!-- line break (self-closing) --> <hr> <!-- horizontal rule (thematic break) -->

3. Lists · ordered & unordered

<ul> <!-- unordered list (bullet) --> <li>first item</li> <li>second item</li> </ul> <ol> <!-- ordered list (numbered) --> <li>step one</li> <li>step two</li> </ol> <!-- nested lists are fine --> <ul> <li>item <ul><li>nested</li></ul> </li> </ul>

Hyperlinks · <a>

<a href="https://example.com" target="_blank">visit example</a> <!-- target="_blank" opens in new tab --> <a href="#section-id">internal jump</a> <!-- anchor link -->

4. Images & media

<img src="photo.jpg" alt="description of image" width="300" height="200"> <!-- alt is mandatory for accessibility --> <video controls width="400"> <source src="video.mp4" type="video/mp4"> your browser doesn't support video. </video> <audio controls> <source src="audio.mp3" type="audio/mpeg"> </audio>

5. Tables · structured data

<table border="1"> <thead> <tr><th>Name</th><th>Age</th></tr> </thead> <tbody> <tr><td>Alice</td><td>28</td></tr> <tr><td>Bob</td><td>32</td></tr> </tbody> <tfoot> <tr><td colspan="2">total: 2 people</td></tr> </tfoot> </table> <!-- colspan / rowspan for merging cells -->

6. Forms · user input

<form action="/submit" method="post"> <label for="name">Name:</label> <input type="text" id="name" name="username" placeholder="your name"> <label>Email: <input type="email" name="email" required></label> <input type="radio" id="male" name="gender" value="male"> <label for="male">Male</label> <select name="city"> <option value="nyc">New York</option> <option value="lon">London</option> </select> <textarea name="message" rows="3" cols="30">default text</textarea> <button type="submit">Send</button> <input type="reset" value="clear"> </form>

7. Semantic HTML · meaning & structure

Use elements that describe what the content is, not just how it looks.

<header> <!-- site header / intro --> <nav> <!-- navigation links --> <main> <!-- main content (only one per page) --> <article> <!-- self-contained content --> <section> <!-- thematic group --> <aside> <!-- sidebar / complementary --> <footer> <!-- footer / credits --> <figure> <!-- figure with figcaption --> <time> <!-- machine-readable date -->

8. Advanced · iframes, data, ARIA

iframe · embed external content

<iframe src="https://example.com" width="400" height="300" title="example"></iframe>

Custom data attributes · data-*

<div data-user-id="123" data-role="admin">user panel</div> <!-- accessible via JS: element.dataset.userId -->

ARIA · accessibility

<nav role="navigation" aria-label="main">...</nav> <button aria-expanded="false" aria-controls="menu">menu</button>

Template & slot (web components)

<template id="card"> <div class="card"> <slot name="title">title</slot> <slot name="content">content</slot> </div> </template>

9. Input types · modern controls

<input type="color"> <!-- color picker --> <input type="date"> <!-- date picker --> <input type="range" min="0" max="100"> <input type="search"> <!-- search field --> <input type="tel"> <!-- telephone --> <input type="url"> <!-- URL --> <input type="number" step="1"> <input type="password" autocomplete="current-password"> <input type="file" accept=".png, .jpg" multiple> <datalist id="browsers"> <!-- autocomplete list --> <option value="chrome"><option value="firefox"> </datalist>

10. Meta tags · SEO & performance

<meta name="description" content="free HTML course"> <meta property="og:title" content="HTML notes"> <meta name="robots" content="index, follow"> <link rel="icon" href="favicon.ico"> <link rel="stylesheet" href="style.css"> <script src="script.js" defer></script> <!-- defer = load after HTML -->

11. HTML entities · reserved characters

&lt; = < &gt; = > &amp; = & &quot; = " &copy; = © &reg; = ® &nbsp; = non-breaking space &mdash; = —

12. Global attributes · all elements

class="..." id="..." style="..." title="..." lang="en" tabindex="0" hidden draggable="true" contenteditable="true" spellcheck="true" translate="no"

13. Interactive elements · details, dialog

<details> <summary>click to expand</summary> <p>hidden content now visible</p> </details> <dialog open> <!-- modal dialog --> <p>this is a dialog box</p> <button onclick="this.closest('dialog').close()">close</button> </dialog>

14. SVG & MathML · vector graphics & math

<svg width="100" height="100"> <circle cx="50" cy="50" r="40" fill="teal" /> </svg> <math> <mi>E</mi> <mo>=</mo> <mi>m</mi> <msup><mi>c</mi><mn>2</mn></msup> </math>

15. Best practices · write clean HTML

  • always use alt for images
  • use semantic elements (<article> instead of <div>)
  • indent consistently (2 or 4 spaces)
  • lowercase tag names and attributes
  • use <label> for form controls
  • validate with W3C validator
